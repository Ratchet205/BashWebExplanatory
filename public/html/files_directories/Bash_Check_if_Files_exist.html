Bash Check if Files exist

Wenn geprüft wird, ob eine Datei existiert, sind die am häufigsten verwendeten FILE-Operatoren -e und -f. Die erste prüft, ob eine Datei existiert, unabhängig vom Typ, während die zweite nur dann true zurückgibt, wenn die FILE eine reguläre Datei ist (kein Verzeichnis oder ein Gerät). Die am besten lesbare Option bei der Überprüfung, ob eine Datei existiert oder nicht, ist die Verwendung des test -Befehls in Verbindung mit der if -Anweisung Jedes der folgenden Snippets überprüft, ob die Datei /etc/resolv.conf existiert:

FILE=/etc/resolv.conf
if test -f "$FILE";
then
echo "$FILE existiert."
fi

Sie können den test-Befehl auch ohne die if-Anweisung verwenden. Der Befehl nach dem &&-Operator wird nur ausgeführt, wenn der Exit-Status des test-Befehls wahr ist:

test -f /etc/resolv.conf && echo "$FILE existiert."

Anstatt komplizierte geschachtelte if/else Konstrukte zu verwenden, können Sie - a (oder && mit [[)) verwenden, um zu testen, ob mehrere Dateien existieren:

if [ -f /etc/resolv.conf -a -f /etc/hosts ];
then
echo "Beide Files existieren."
fi
