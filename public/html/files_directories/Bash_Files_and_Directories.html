Bash Files & Directories
Print Current Working Directory (pwd)
Ihr aktuelles Arbeitsverzeichnis ist das Verzeichnis, in dem Ihre Befehle ausgeführt werden. Es wird normalerweise als der vollständige Pfad zum Verzeichnis ausgegeben (was bedeutet, dass Sie das übergeordnete Verzeichnis sehen können). Um den Namen des aktuellen Arbeitsverzeichnisses auszudrucken, verwenden Sie den Befehl pwd.

    
    Guest@BashStruktur~$ pwd
                         /users/schuster


 
Change Current Working Directory (cd)
Oft möchten Sie vielleicht das aktuelle Arbeitsverzeichnis ändern, so dass Sie auf verschiedene Unterverzeichnisse und Dateien zugreifen können. Um Verzeichnisse zu ändern, verwenden Sie den Befehl cd gefolgt von dem Namen des Verzeichnisses (z.B. cd downloads). Anschließend können Sie Ihr aktuelles Arbeitsverzeichnis erneut drucken, um den neuen Pfad zu überprüfen.

    
    Guest@BashStruktur~$ cd documents
    Guest@BashStruktur~$ pwd
                         /users/schuster/documents



You can go back to the parent directory of any current directory by usingthe command cd .., as the full path of the current working directory isunderstood by Bash.

    
    Guest@BashStruktur~$ cd..
    Guest@BashStruktur~$ pwd
                         /users/schuster


 
Create a New Directory (mkdir)
Der erste Schritt beim Erstellen eines neuen Verzeichnisses besteht darin, mit cd zu dem Verzeichnis zu navigieren, das Sie als übergeordnetes Verzeichnis des neuen Verzeichnisses verwenden möchten. Dann verwenden Sie den Befehl mkdir gefolgt von dem Namen, den Sie dem neuen Verzeichnis geben möchten (z.B. mkdir directory-name).

    
    Guest@BashStruktur~$ cd documents
    Guest@BashStruktur~$ mkdir assignments
    Guest@BashStruktur~$ cd assignments
    Guest@BashStruktur~$ pwd
                         /users/schuster/documents/assignments


 
Print a List of Files and Subdirectories (ls)
Um eine Liste aller Unterverzeichnisse und Dateien innerhalb des aktuellen Arbeitsverzeichnisses zu sehen, verwenden Sie den Befehl ls.

    
    Guest@BashStruktur~$ cd ~
    Guest@BashStruktur~$ pwd
                         /users/schuster
    Guest@BashStruktur~$ ls
                         addresse.txt   documents   downloads   noten.txt                   


 
Delete a File (rm)
Um eine bestimmte Datei zu löschen, verwenden Sie den Befehl rm gefolgt von dem Namen der Datei, die Sie löschen möchten (z.B. rm filename).

    
    Guest@BashStruktur~$ pwd
                         /users/schuster
    Guest@BashStruktur~$ ls
                         addresse.txt   documents   downloads   noten.txt
    Guest@BashStruktur~$ rm addresse.txt   
    Guest@BashStruktur~$ ls
                         documents   downloads   noten.txt                               


 
Delete a Directory (rm -r)
Um ein Verzeichnis und alle darin enthaltenen Unterverzeichnisse und Dateien zu löschen (d.h. zu entfernen), navigieren Sie zum übergeordneten Verzeichnis und verwenden dann den Befehl rm -r gefolgt von dem Namen des zu löschenden Verzeichnisses (z.B. rm -rdirectory-name).

    
    Guest@BashStruktur~$ cd ~
    Guest@BashStruktur~$ cd documents
    Guest@BashStruktur~$ pwd
                         /users/schuster/documents
    Guest@BashStruktur~$ ls
                         assignments
    Guest@BashStruktur~$ rm -r assignments                    



Das rm steht für remove, während das -r notwendig ist, um Bash mitzuteilen, dass es den Befehl durch eine Liste aller Dateien und Unterverzeichnisse innerhalb des übergeordneten Verzeichnisses wiederholen muss. Somit wird auch das neu erstellte Homeworks-Verzeichnis unter Zuordnungen gelöscht, wenn Zuordnungen gelöscht werden.

    
    Guest@BashStruktur~$ pwd
                         /users/schuster/documents
    Guest@BashStruktur~$ ls
    Guest@BashStruktur~$ 


 
Copy a File (cp)
Sie können auch eine bestimmte Datei in ein neues Verzeichnis kopieren, indem Sie den Befehl cp gefolgt von dem Namen der Datei, die Sie kopieren möchten, und dem Namen des Verzeichnisses, in das Sie die Datei kopieren möchten (z.B. cp filename directory-name).

    
    Guest@BashStruktur~$ cd ~
    Guest@BashStruktur~$ pwd
                         /users/schuster
    Guest@BashStruktur~$ ls
                         documents   downloads   noten.txt
    Guest@BashStruktur~$ cp noten.txt documents  
    Guest@BashStruktur~$ cd documents 
    Guest@BashStruktur~$ ls
                         noten.txt                             


 
Copy a Directory and Its Contents (cp -r)
Ebenso können Sie ein ganzes Verzeichnis in ein anderes Verzeichnis kopieren, indem Sie cp -r gefolgt von dem Verzeichnisnamen, das Sie kopieren möchten, und dem Namen des Verzeichnisses, in das Sie das Verzeichnis kopieren möchten (z.B. cp -r directory-name-1 directory-name-2). Ähnlich wie rm -r ist auch -r in cp -r notwendig, um Bash mitzuteilen, dass er den Befehl durch eine Liste aller Dateien und Unterverzeichnisse innerhalb des übergeordneten Verzeichnisses wiederholen muss.

    
    Guest@BashStruktur~$ cd ~
    Guest@BashStruktur~$ pwd
                         /users/schuster
    Guest@BashStruktur~$ ls
                         documents   downloads   noten.txt
    Guest@BashStruktur~$ mkdir bilder
    Guest@BashStruktur~$ ls
                         documents   downloads   noten.txt  bilder
    Guest@BashStruktur~$ cp -r bilder documents  
    Guest@BashStruktur~$ cd documents 
    Guest@BashStruktur~$ ls
                         noten.txt  bilder                            


 
Create a New File Using a Single Command (touch)
Sie können eine neue leere Datei mit dem einfachen Befehl touch erstellen (z.B. touch file-name.txt). Dieser Befehl wurde ursprünglich erstellt, um die Zeitstempel von Dateien zu verwalten. Wenn jedoch eine Datei noch nicht existiert, erstellt der Befehl die Datei. Dies ist eine unglaublich nützliche Möglichkeit, schnell und programmatisch eine neue leere Datei zu erstellen, die zu einem späteren Zeitpunkt gefüllt werden kann.

    
    Guest@BashStruktur~$ cd ~
    Guest@BashStruktur~$ cd downloads
    Guest@BashStruktur~$ pwd
                         /users/schuster/downloads
    Guest@BashStruktur~$ touch veg-data.txt
    Guest@BashStruktur~$ ls
                         veg-data.txt
