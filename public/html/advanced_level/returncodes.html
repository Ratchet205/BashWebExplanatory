Returncodes auf Linux

Returncodes, auch bekannt als Exitcodes oder Exit-Status, sind numerische Werte, die ein Linux-Programm zurückgibt, wenn es beendet wird. Diese Codes geben an, ob das Programm erfolgreich ausgeführt wurde oder ob es während der Ausführung auf Probleme gestoßen ist. Ein Returncode von 0 zeigt an, dass das Programm erfolgreich beendet wurde, während andere Werte auf verschiedene Fehler oder Situationen hinweisen.

    Die wichtigsten Returncodes:
  • 0: Erfolgreiche Ausführung ohne Fehler.
  • 1: Allgemeiner Fehler. Dies wird oft verwendet, wenn keine spezifische Information verfügbar ist.
  • 2: Fehler aufgrund unzulässiger Befehlszeilenargumente.
  • 126: Das aufgerufene Programm konnte nicht ausgeführt werden. Dies kann auf Berechtigungsprobleme hinweisen.
  • 127: Das aufgerufene Programm wurde nicht gefunden.
  • 128 + N: Das Programm wurde durch ein Signal N beendet (z. B. 128 + 9 für SIGKILL).
  • 130: Das Programm wurde durch Benutzerabbruch (Ctrl+C) beendet.
  • 255: Ungültiger Exit-Code.


Beispiele:

 if [ $? -eq 0 ];
 then 
     echo "Erfolgreich";
 else 
     echo "Fehler";
 fi             
 #!/usr/bin/expect
 # Das Programm aufrufen
 spawn ./mein_programm

 # Warten auf das Ende des Programms
 expect eof

  # Den Returncode überprüfen
 set return_code [wait]

 if {[lindex $return_code 3] == 0} {
     puts "Das Programm wurde erfolgreich ausgeführt."
 } else {
     puts "Beim Ausführen des Programms ist ein Fehler aufgetreten."
 }
